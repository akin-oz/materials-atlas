# Citing Materials Atlas

Materials Atlas is intended to be reused, adapted, taught from, and improved.

Citation matters because it preserves attribution for the editorial work behind the repository: the learning paths, conceptual structure, diagrams, references, and synthesis that make the atlas useful.

---

# How To Cite

If you use Materials Atlas in a paper, course, workshop, repository, presentation, or research project, cite the repository.

Use the citation metadata in [CITATION.cff](CITATION.cff) whenever possible. GitHub can read this file and generate citation formats automatically.

Until the first stable release receives a DOI, cite the GitHub repository:

```text
Oztorun, A. Materials Atlas. GitHub repository. https://github.com/akin-oz/materials-atlas
```

---

# BibTeX

```bibtex
@misc{materials_atlas,
  author       = {Akin Oztorun},
  title        = {Materials Atlas},
  howpublished = {GitHub repository},
  url          = {https://github.com/akin-oz/materials-atlas},
  note         = {Open knowledge repository for Computational Materials Science}
}
```

---

# Future DOI

After the first stable public release, Materials Atlas is expected to be archived through Zenodo.

At that point, this guide and [CITATION.cff](CITATION.cff) will be updated with the DOI.

Prefer the DOI citation once it exists.

# Attribution Expectations

Materials Atlas is licensed under CC BY 4.0.

You may copy, redistribute, remix, transform, and build upon the repository content, including for commercial purposes, provided that appropriate attribution is given.

Good attribution includes:

- the project name: Materials Atlas
- the author: Akin Oztorun
- a link to the repository
- a link to the CC BY 4.0 license
- a note if you modified the material

For example:

```text
Adapted from Materials Atlas by Akin Oztorun, licensed under CC BY 4.0.
https://github.com/akin-oz/materials-atlas
```

Attribution should be clear, durable, and easy for readers to follow.
