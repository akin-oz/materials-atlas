# Study Method

> A repeatable workflow for learning Computational Materials Science.

This curriculum is competency-based.

The goal is not to finish books.

The goal is to build durable scientific intuition.

Every study session should produce understanding and one durable artifact.

---

# The Learning Loop

Every study session follows the same four-step process.

```
Read
    ↓
Reflect
    ↓
Build
    ↓
Record
```

Do not skip steps.

---

# Step 1 — Read

Spend approximately 30–45 minutes reading.

The objective is understanding, not completion.

Do not take detailed notes while reading.

Instead, ask:

- What is the central idea?
- Why does this matter?
- What problem does this concept solve?

---

# Step 2 — Reflect

Pause for a few minutes.

Answer only three questions.

## Big Idea

What is the single most important idea?

## Computational View

How would a computer represent this concept?

## Future Connection

Where will this idea appear again later?

If you cannot answer these questions, continue reading before moving on.

---

# Step 3 — Build

Produce exactly one artifact.

Choose only one.

Examples:

- Jupyter notebook
- Python script
- visualization
- markdown note
- reproduced figure
- paper summary

Do not produce multiple artifacts in one session.

The purpose is reinforcement, not productivity.

---

# Step 4 — Record

Record what survives.

Do not summarize the entire chapter.

Capture only the ideas worth remembering.

A chapter note should answer:

## Big Idea

One paragraph.

## Why It Matters

Why does this concept exist?

## Computational View

How is this represented computationally?

## Related Topics

Which future modules depend on it?

## Artifact

Link to the notebook, script, or visualization.

## Open Questions

What is still unclear?

---

# Weekly Review

Once each week, spend 20–30 minutes reviewing.

Do not reread chapters.

Instead, ask:

- What new mental models did I build?
- Which concepts still feel weak?
- What should I study next?

Update notes only if understanding has improved.

---

# One Artifact Rule

Every study session should produce at most one durable artifact.

Examples:

- one notebook
- one Python script
- one visualization
- one markdown note

If a session produces five artifacts, the work was probably fragmented.

If a session produces none, the learning probably will not last.

---

# Reading Philosophy

Never read passively.

Never optimize for finishing books.

Never memorize equations before understanding concepts.

Prefer:

- understanding over coverage
- intuition over memorization
- implementation over repetition
- consistency over intensity

---

# ChatGPT Review

After finishing a chapter or major topic, explain it without looking at your notes.

Use ChatGPT as an oral examiner.

Prefer questions that test reasoning instead of memory.

Examples:

- Why is this concept important?
- How would you implement it?
- Where will it appear later?
- What assumptions does it make?
- What would fail if it were removed?

If you can teach the idea, you probably understand it.

---

# Completion Rule

A chapter is complete when:

- you understand its central idea
- you created one durable artifact
- you can explain it in your own words
- you know where it fits in the larger field

Only then move to the next topic.
