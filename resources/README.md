# Resources

Resources are secondary indexes.

They are not the primary navigation of Materials Atlas. Meaning lives first in domains.

## Purpose

The purpose of `resources/` is to index important materials across the atlas once there is enough curated content to justify the index.

Use [../EDITORIAL.md](../EDITORIAL.md) for resource justification and relationship vocabulary.

The first justified indexes are:

- [books/](books/) for durable books.
- [papers/](papers/) for paper selection philosophy and starter anchors.
- [software/](software/) for foundational software infrastructure.
- [videos/](videos/) for high-value long-form learning material.

Other resource types may be added later only when they improve retrieval.

## Scope

`resources/` should help readers find included resources across domains.

It should not become a dumping ground for links. A resource belongs here only when it supports understanding of one or more domains.

## Inclusion Criteria

Resource inclusion is governed by [../EDITORIAL.md](../EDITORIAL.md). A resource belongs here only when it supports understanding of one or more domains and its importance can be explained clearly.

## Relationship To Domains

Resources do not define the map.

A paper matters because of the domain it changed. A software package matters because of where it is used. A dataset matters because of the questions it enables. A course matters because of the learning path it supports.

For that reason, resource indexes should point back to domain pages whenever possible.

## When To Add New Pages

Do not create additional resource-type subdirectories until enough curated resources exist to justify them.

For example, do not create separate indexes for datasets, courses, conferences, benchmarks, or standards until those indexes would contain real curated entries and improve retrieval.
