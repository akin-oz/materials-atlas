# Domains

Domains are the primary organizing principle of Materials Atlas.

A domain is a durable area of Computational Materials Science or a closely connected field. It may describe a method, material class, research program, computational scale, or scientific problem area.

No domain pages are created yet. A domain page should be added only when there is enough curated content to make the page useful immediately.

## Purpose

The purpose of `domains/` is to organize knowledge around the way researchers and learners actually navigate the field.

A reader usually asks:

- I want to understand density functional theory.
- I want to study materials informatics.
- I want to enter battery materials.
- I want to understand AI for materials.
- I want to know which tools and papers matter in this area.

Those questions are domain-centered, not resource-type-centered.

## Scope

Future domain pages may cover areas such as computational methods, materials classes, modeling scales, discovery workflows, informatics methods, or research themes.

Domain pages should connect concepts, papers, software, datasets, people, institutions, applications, and open problems into a coherent map.

They should not be exhaustive encyclopedic articles.

## Canonical Template

Every domain page must start from [TEMPLATE.md](TEMPLATE.md).

The template defines the canonical structure for future domains. It keeps domain pages comparable without turning them into a schema or database.

## When To Add New Pages

Add a new page only when the repository has real curated content for that domain and the page reduces uncertainty for a reader.

If a domain page would contain only placeholders, do not create it.
