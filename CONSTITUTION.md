# Materials Atlas Constitution

> Durable principles for repository decisions.

## Purpose

This document defines the principles that remain stable when implementation, tooling, and content change. It does not define editorial mechanics; see [EDITORIAL.md](EDITORIAL.md) for those.

## Principles

1. Knowledge before volume. A smaller, clearer atlas is more useful than a comprehensive-looking collection.
2. Content earns structure. New pages, directories, and abstractions require evidence that they reduce uncertainty.
3. One canonical explanation. Reuse a durable reference rather than maintaining parallel explanations.
4. Domains synthesize. A domain connects concepts, evidence, tools, and context; it is never a placeholder or link list.
5. References define. Canonical references own stable explanations, notation, and reusable diagrams.
6. Roadmaps teach. Learning modules define progression and capability, not encyclopedic coverage.
7. Resources remain curated. External work belongs only when its role in the field can be explained.
8. Diagrams are editorial artifacts. Every canonical diagram has an owner, a purpose, and a reuse context.
9. Evidence over assertion. Scientific and ecosystem claims should be traceable to primary or durable sources.
10. Maintenance is a feature. Delete, merge, or clarify material when doing so improves conceptual navigation.

## Use

When a proposal conflicts with these principles, revise or reject it. For repository layers and information flow, see [ARCHITECTURE.md](ARCHITECTURE.md). For scientific boundaries, see [SCIENTIFIC_SCOPE.md](SCIENTIFIC_SCOPE.md).
